# 기획 측면 개선 제안서 - 개인 기술 블로그 & 취업용 포트폴리오

> 작성일: 2025-07-10  
> 작성자: 기획자

## 📋 개요

본 제안서는 개인 기술 블로그 및 취업용 포트폴리오 사이트의 기획 측면 개선 방향을 제시합니다. 특히 /about 페이지를 중심으로 채용 담당자와 기술 리더들에게 효과적으로 어필할 수 있는 전략을 수립했습니다.

---

## 1. /about 페이지 개선 방향

### 1.1 페이지 구조 재설계

#### 현재 문제점
- 단순 나열식 정보 제공으로 임팩트 부족
- 기술 스택이 아이콘으로만 표시되어 숙련도 파악 어려움
- 프로젝트와 경력 간의 연관성 부각 부족
- "작업 진행중" 안내 문구가 신뢰도 저하

#### 개선 방향

**1) 히어로 섹션 강화**
```
[프로필 사진] → [핵심 가치 제안] → [Quick Stats]
              "3년차 프론트엔드 개발자에서 
               기술과 비즈니스를 연결하는 PM으로"
               
               • 3년 실무 경력
               • 15+ 프로젝트 완수
               • PM 부트캠프 수료
```

**2) 스토리텔링 도입**
- 개발자에서 PM으로의 전환 스토리
- 각 경력별 주요 성과와 학습 포인트
- 프로젝트별 문제 해결 사례

### 1.2 콘텐츠 계층 구조 최적화

```
1. 간단한 자기소개 (Elevator Pitch)
   └─ 30초 안에 나를 설명하는 핵심 메시지

2. 핵심 역량 (Core Competencies)
   ├─ 기술적 역량 (개발 스킬)
   ├─ PM 역량 (기획, 관리)
   └─ 소프트 스킬 (커뮤니케이션, 리더십)

3. 경력 하이라이트 (Career Highlights)
   └─ 각 경력별 Impact 중심 서술

4. 프로젝트 포트폴리오
   └─ 역할, 기여도, 성과 명확히 표시

5. 성장 스토리 (Growth Journey)
   └─ 지속적인 학습과 성장 의지 표현
```

### 1.3 기술 스택 표현 개선

**현재**: 단순 아이콘 나열

**개선안**: 
```
[기술명] + [숙련도] + [실무 경험]

예시:
React ●●●●○ (3년)
- 탑툰 글로벌 서비스 구축
- 성능 최적화로 로딩 속도 40% 개선
- 컴포넌트 라이브러리 구축
```

## 2. 블로그 콘텐츠 전략

### 2.1 콘텐츠 카테고리 재정립

#### 기술 블로그
- **실무 경험 시리즈**: 실제 프로젝트에서 마주친 문제와 해결 과정
- **기술 깊이 탐구**: 특정 기술에 대한 심도 있는 분석
- **튜토리얼**: 주니어 개발자를 위한 가이드

#### PM 인사이트
- **프로젝트 회고**: PM 관점에서의 프로젝트 분석
- **기획 프로세스**: 실제 기획 경험 공유
- **팀 협업 사례**: 개발자 출신 PM의 강점 활용 사례

### 2.2 콘텐츠 품질 관리

**콘텐츠 작성 가이드라인**
1. 명확한 문제 정의
2. 구체적인 해결 과정
3. 측정 가능한 결과
4. 얻은 교훈과 인사이트

---

## 3. 포트폴리오 프레젠테이션 개선

### 3.1 프로젝트 카드 정보 구조화

```
[프로젝트명]
├─ 역할: PM / Frontend Developer
├─ 기간: 2024.06 - 2024.08
├─ 팀 구성: PM 1, 개발자 3, 디자이너 1
├─ 핵심 기여
│  ├─ 요구사항 정의서 작성
│  ├─ 스프린트 계획 및 진행
│  └─ 이해관계자 커뮤니케이션
└─ 성과
   ├─ 일정 내 출시 완료
   └─ 사용자 만족도 85% 달성
```

### 3.2 사례 연구(Case Study) 도입

각 주요 프로젝트에 대한 상세 페이지:
1. **배경과 문제**: 프로젝트 시작 계기
2. **접근 방법**: 문제 해결 프로세스
3. **실행 과정**: 주요 의사결정과 근거
4. **결과와 영향**: 정량적/정성적 성과
5. **학습과 성장**: 개인적 성장 포인트

## 4. 사용자 경험 최적화

### 4.1 네비게이션 개선

**빠른 이동 메뉴**
- About (이력서)
- Projects (포트폴리오)
- Blog (기술 블로그)
- Contact (연락처)

**스크롤 진행 표시기**
- 긴 페이지에서 현재 위치 표시
- 섹션별 빠른 이동 버튼

### 4.2 인터랙션 디자인

**마이크로 인터랙션**
- 호버 시 추가 정보 표시
- 부드러운 스크롤 애니메이션
- 로딩 상태 시각화

**반응형 최적화**
- 모바일에서도 읽기 편한 레이아웃
- 터치 친화적 인터페이스

## 6. 성과 측정 지표

### 6.1 정량적 지표
- 페이지 체류 시간
- 스크롤 깊이
- Contact 클릭률
- 프로젝트 상세 페이지 조회수

- 기술 커뮤니티 인지도

## 7. 실행 우선순위

### Phase 1 (1-2주)
1. /about 페이지 "작업 진행중" 제거
2. 경력 섹션 Impact 중심 재작성
3. 기술 스택 숙련도 표시 추가

### Phase 2 (3-4주)
1. 프로젝트 상세 페이지 구축
2. 사례 연구 1-2개 작성
3. 네비게이션 UX 개선

### Phase 3 (5-6주)
1. 블로그 콘텐츠 카테고리 재정립
2. SEO 최적화
3. 성과 측정 도구 도입

## 결론

개인 기술 블로그와 취업용 포트폴리오의 핵심은 "나만의 차별화된 가치"를 명확하게 전달하는 것입니다. 개발자 출신 PM이라는 독특한 포지션을 강점으로 부각시키고, 실무 경험과 지속적인 성장 스토리를 통해 신뢰를 구축해야 합니다.

특히 /about 페이지는 첫인상을 결정하는 중요한 페이지로, Impact 중심의 스토리텔링과 시각적으로 매력적인 정보 구조화가 필수적입니다.
